Design constraints:

- This project is for demonstrating senior distributed system architecture ability.
- Do not turn it into a generic admin dashboard.
- Do not replace the architecture views with plain cards or charts.
- Do not overuse decorative UI that weakens technical clarity.
- Prioritize architecture readability, simulation logic, and professional presentation quality.
- The simulator should feel credible to experienced engineers.

Use concise enterprise architecture wording in the UI.

Examples:

- Local Transaction Boundary
- Event Dispatch
- Async Processing
- Failure Handling
- Metrics & Alerting
- Root Cause Isolation
- Business Recoverability
- Ingress Control
- Elastic Service Layer
- Downstream Protection

Avoid childish or generic wording like:

- cool animation
- fun demo
- magic flow
- awesome system
